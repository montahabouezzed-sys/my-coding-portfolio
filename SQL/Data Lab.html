Vehicle Sensor Analysis (SQL + Python)

Project Overview

This project demonstrates a complete data pipeline. I modeled an automotive database in DuckDB, processed sensor readings with Python, and generated time-series visualizations to identify vehicle anomalies.

🚀 View Full GitHub Repository

1. Database Schema & Modeling

The database is structured to handle high-frequency sensor logs. Below is the primary table definition used for storage:


            CREATE TABLE sensor_readings (
            reading_id INTEGER PRIMARY KEY,
            test_run_id INTEGER,
            timestamp DATETIME,
            sensor_type TEXT,
            value REAL,
            FOREIGN KEY (test_run_id) REFERENCES test_runs(test_run_id)
        );

2. Data Analysis with Python

Using Pandas and DuckDB, I extracted the raw data and applied a rolling average to smooth out sensor noise.


import duckdb
import pandas as pd

# Query data into a DataFrame
con = duckdb.connect('../automotive.duckdb')
df = con.sql("SELECT * FROM sensor_readings").df()

# Calculate moving average
df['moving_avg'] = df.groupby('sensor_type')['value'].transform(lambda x: x.rolling(window=5).mean())
