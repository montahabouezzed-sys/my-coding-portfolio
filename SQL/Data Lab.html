Vehicle Sensor Analysis (SQL + Python)

A complete data pipeline for modeling, querying, and analyzing automotive sensor data. This project simulates a realistic engineering workflow used in vehicle testing, anomaly detection, and sensor reliability analysis.

1. Database Schema & Modeling

The relational schema is designed to handle high‑frequency sensor logs from multiple vehicle test runs. Below is the core table used for storing timestamped sensor values: 


CREATE TABLE sensor_readings (
    reading_id INTEGER PRIMARY KEY,
    test_run_id INTEGER,
    timestamp DATETIME,
    sensor_type TEXT,
    value REAL,
    FOREIGN KEY (test_run_id) REFERENCES test_runs(test_run_id)
);

This structure supports time‑series analysis, anomaly detection, and efficient querying using DuckDB.

2. Data Analysis with Python

Using Pandas and DuckDB, I extracted raw sensor logs, grouped them by sensor type, and applied rolling statistics to smooth noise and detect abnormal behavior. 


import duckdb
import pandas as pd

# Query data into a DataFrame
con = duckdb.connect('../automotive.duckdb')
df = con.sql("SELECT * FROM sensor_readings").df()

# Calculate moving average per sensor type
df['moving_avg'] = (
    df.groupby('sensor_type')['value']
      .transform(lambda x: x.rolling(window=5).mean())
)
